## ğŸ“ Documentation Guidelines

### File Naming

- Use lowercase with hyphens: `quick-start.md`
- Suffix by type:
  - `-report.md` for diagnostic reports
  - `-analysis.md` for technical analysis
  - `-summary.md` for executive summaries
  - `-guide.md` for how-to guides

### Structure

Each document should include:
1. **Clear title** describing the content
2. **Problem statement** (if applicable)
3. **Solution/approach**
4. **Code examples** (if applicable)
5. **Related documents** section
